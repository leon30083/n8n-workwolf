这是一个非常强大的工作流程。您所描述的（将前一个镜头的最后一帧用作下一个镜头的`input_reference`）是专业人士用来“强制”实现场景和风格一致性的核心技术 [1, 2]。

正如我们的研究所表明的，当您使用`input_reference`（图像输入）时，API的逻辑会发生根本性的变化。模型将参考图像用作“第一帧的锚点”，而您的**文本提示词则定义了接下来会发生什么** [3]。

因此，一个“优秀”的提示词（在您的工作流程中）**不应该**浪费词语去描述图像中*已经存在*的内容（例如“一个穿着红色斗篷的女孩”）。相反，它应该*只*专注于**动作、摄像机运动和声音**。

这里有一份专为您这种“图像转视频”工作流设计的优秀提示词范例。

---

### 情景：动画角色的多镜头序列

这个例子假设您的目标是创建一个连贯的叙事序列，主角是一个**动画风格**的角色。

**重要提示：** 这个例子使用的是*动画*角色，因为API的内容策略明确*禁止*使用包含**逼真人类面孔**的参考图像 [4, 5, 6, 7, 8]。您的工作流程对于非人类主体（如动画角色、动物、产品）是完美且合规的。

#### API 调用 1：建立镜头（文生视频）

您首先需要一个“起始镜头”来建立您的角色和场景。

> **提示词 (Prompt):**
> “宫崎骏风格的动画，电影感。一个娇小、胆怯的森林精灵，全身散发着柔和的蓝色光芒，有着一双大大的好奇眼睛。
>
> **场景:** 它躲在一个长满苔藓的巨大树根后面，偷偷向外看。这是一个黄昏时分的魔法森林地面，体积光（'上帝之光'）穿过树冠照射下来 [3, 9, 10, 11, 12]。
>
> **摄像机:** 85mm 镜头，浅景深 (f/2.2)，摄像机固定 [3, 13, 14, 15]。
>
> **动作:** 森林精灵的眼睛眨了两次，它的头轻微向左倾斜。
>
> **音频:** 环境森林声，轻柔的风声，微弱的魔法嗡嗡声 [16]。”

*   **您的操作：** 您运行此API调用。Sora 2生成一个4秒的视频。您的工具截取此视频的**最后一帧**（例如，`frame_01.png`）。这一帧现在是您下一个镜头的视觉参考。

---

#### API 调用 2：延续镜头（图生视频）

现在，您使用您的工具将`frame_01.png`作为`input_reference` [3] 发送，并配上**这个专门设计的提示词**：

> **提示词 (Prompt):**
> “保持已建立的宫崎骏风格和浅景深。
>
> **动作:** 精灵小心翼翼地从树根后面走出来，它好奇地歪着头向上看。它向右迈出了三小步，踩在了一个发光的蘑菇上，蘑菇在被触碰的瞬间变得更亮 [3, 9]。
>
> **摄像机:** 摄像机保持静止。
>
> **音频:** 苔藓上轻柔的脚步声，当蘑菇变亮时发出‘叮’的魔法铃声 [17, 16, 18, 19]。”

---

### 技术说明：为什么这个提示词是“优秀”的

这个提示词之所以在您的工作流程中表现出色，原因如下：

1.  **它只描述“下一步行动”**：
    提示词没有浪费任何文字去描述“一个蓝色精灵”或“在森林里”。`input_reference` 图像已经为模型提供了这些信息。提示词完全专注于**动作**（“走出来”、“踩上去”、“变亮”），这正是API所期望的 [3]。

2.  **它使用“节拍” (Beats) 来描述动作**：
    提示词没有说“精灵走路”，而是说“迈出了三小步”。根据官方的提示词指南，Sora 2对这种基于“节拍”或“计数”的精确动作描述（如“骑车人踩了三下踏板，刹车，停下”）反应最好 [3, 9]。这能产生更可控、更逼真的物理效果。

3.  **它强化了“风格化脊柱” (Style Spine)**：
    尽管图像输入锚定了风格，但最佳实践是*强化*关键的风格元素。通过包含“保持已建立的宫崎骏风格和浅景深”，您确保模型不会“即兴创作”并突然改变镜头类型或美学 [3, 1]。

4.  **音频提示词增强了物理一致性**：
    Sora 2 是一个联合的音视频模型 [20, 21, 17, 22, 7]。通过明确要求“苔藓上轻柔的脚步声”和“叮的魔法铃声”，您实际上是*迫使*模型在视觉上渲染导致这些声音的物理事件（脚接触苔藓，蘑菇被激活）。这是一种确保音视频同步和增强场景真实感的强大技术。

### 最终警告（重复）

请记住，您工具的有效性高度依赖于输入图像的内容。如果您尝试使用**逼真的人类面孔**作为参考，API调用很可能会失败 [4, 5, 6, 7, 8]。对于动画、品牌吉祥物、产品和动物，此工作流程是实现一致性的黄金标准。
